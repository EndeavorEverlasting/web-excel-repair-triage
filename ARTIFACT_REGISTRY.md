# Artifact Registry

## What Artifacts This Repo Produces

| Artifact Type | Description | Location | Naming Convention |
|--------------|-------------|----------|-------------------|
| Repaired Workbooks | Fixed .xlsx files | `Repaired/` | `{original_name}_repaired_{timestamp}.xlsx` |
| Forensic Reports | Repair analysis reports | `Outputs/` | `{original_name}_forensic_{timestamp}.html` |
| Patch Recipes | JSON patch instructions | `Outputs/` | `{original_name}_patch_{timestamp}.json` |
| Sidecar Files | Metadata sidecars | `Outputs/` | `{original_name}_sidecar_{timestamp}.json` |
| Admin Sheets | Admin-facing Project Team sheets | `Outputs/` | `{original_name}_admin_{timestamp}.xlsx` |
| Task Context | Task tracker context files | `Outputs/` | `{original_name}_context_{timestamp}.json` |
| Test Reports | Test execution reports | `Outputs/` | `test_report_{timestamp}.html` |
| Backups | Timestamped backups | `Outputs/backups/` | `{original_name}_backup_{timestamp}.xlsx` |

## Where They Live

| Directory | Purpose | Contents |
|-----------|---------|----------|
| `Repaired/` | Final repaired workbooks | Clean .xlsx files ready for use |
| `Outputs/` | Generated artifacts | Reports, recipes, sidecars, admin sheets |
| `Outputs/backups/` | Timestamped backups | Pre-mutation backups |
| `Outputs/test_reports/` | Test execution reports | HTML test reports |
| `Candidates/` | Read-only inputs | NEVER write here |
| `References/` | Reference materials | Schemas, templates, examples |

## How to Generate Them

### Repaired Workbooks
```bash
# Using the main application
streamlit run app.py
# Then use the web interface to repair a workbook

# Using scripts
python scripts/repair_workbook.py input.xlsx
```

### Forensic Reports
```bash
# Generate forensic analysis
python scripts/generate_forensic.py input.xlsx
# Output: Outputs/input_forensic_YYYYMMDD_HHMMSS.html
```

### Patch Recipes
```bash
# Generate patch recipe
python scripts/generate_patch.py input.xlsx
# Output: Outputs/input_patch_YYYYMMDD_HHMMSS.json
```

### Admin Sheets
```bash
# Roster to admin submission
python scripts/roster_to_admin_submission.py roster.xlsx
# Output: Outputs/roster_admin_YYYYMMDD_HHMMSS.xlsx
```

### Task Context
```bash
# Roster to task context
python scripts/roster_to_task_context.py roster.xlsx
# Output: Outputs/roster_context_YYYYMMDD_HHMMSS.json
```

## Naming Conventions

### General Pattern
```
{artifact_type}_{source_name}_{timestamp}.{extension}
```

### Timestamp Format
```
YYYYMMDD_HHMMSS
```

### Examples
```
repaired_MyWorkbook_20260720_143022.xlsx
forensic_MyWorkbook_20260720_143022.html
patch_MyWorkbook_20260720_143022.json
admin_RosterLog_20260720_143022.xlsx
context_RosterLog_20260720_143022.json
```

## Artifact Lifecycle

1. **Creation** - Artifact is generated by a script or application
2. **Validation** - Artifact is validated against expected format
3. **Storage** - Artifact is stored in the appropriate directory
4. **Tracking** - Artifact is tracked in git (if appropriate)
5. **Cleanup** - Old artifacts are cleaned up per retention policy

## Retention Policy

| Artifact Type | Retention | Cleanup Command |
|--------------|-----------|-----------------|
| Repaired Workboxes | 30 days | `find Repaired/ -mtime +30 -delete` |
| Forensic Reports | 90 days | `find Outputs/ -name "*forensic*" -mtime +90 -delete` |
| Patch Recipes | 180 days | `find Outputs/ -name "*patch*" -mtime +180 -delete` |
| Backups | 7 days | `find Outputs/backups/ -mtime +7 -delete` |
| Test Reports | 30 days | `find Outputs/ -name "*test_report*" -mtime +30 -delete` |

## Safety Rules

1. **Never overwrite source artifacts** - Always create new files with timestamps
2. **Backup before mutation** - Create backup in `Outputs/backups/` before any changes
3. **Validate before delivery** - Run validation checks before delivering artifacts
4. **Track in git** - Add important artifacts to git tracking
5. **Clean up regularly** - Follow retention policy for artifact cleanup
